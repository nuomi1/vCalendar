## Why

现有 `utils.ts` 中的 `inferMarket` 和 `inferInstrumentType` 函数存在以下问题：
1. **北交所规则不完整**：当前仅匹配 `8` 和 `4` 开头，但实际规则是 `92` 开头（股票）和 `810` 开头（可转债）
2. **缺少异常处理**：未匹配规则时返回默认的 `SZ`，可能导致数据混淆
3. **需要 category 参数**：调用时需要传入 category 参数，不够简洁
4. **规则不完整**：上交所/深交所的完整规则未覆盖

## What Changes

1. **移除 category 参数** - inferMarket(code) 和 inferInstrumentType(code) 仅需 code
2. **完整匹配规则** - 上交所/深交所/北交所的所有规则
3. **异常处理** - 未匹配任何规则时抛出 Error(`无法识别的证券代码: ${code}`)
4. **更新调用点** - createJSON、createICS 移除 category 参数

## Match Order

1. **交易所顺序**：上交所 → 深交所 → 北交所
2. **标的类型顺序**：股票 → 可转债 → REITs
3. **同一标的类型内**：按数字从小到大顺序判断
